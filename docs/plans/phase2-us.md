# Phase 2 RAG 能力增强 — 详细用户故事

> 阶段目标：提升 RAG 问答效果和召回质量
> 整理时间：2026-04-14

---

## 1. 文档解析增强

### US-2.1: PDF 文档 OCR 识别
**作为** 开发者  
**我想要** 系统能够识别 PDF 中的图片文字  
**以便** 从扫描版 PDF 或图片型 PDF 中提取内容

**验收标准：**
- [ ] 集成 Tesseract OCR 引擎
- [ ] 自动检测 PDF 中的图片页
- [ ] OCR 识别结果正确提取为文本
- [ ] 原始文本 + OCR 文本合并存储
- [ ] 配置项控制是否启用 OCR

**技术方案：**
```xml
<!-- 引入 Tess4J -->
<dependency>
    <groupId>net.java.dev.tess4j</groupId>
    <artifactId>tess4j</artifactId>
    <version>5.7.0</version>
</dependency>
```

---

### US-2.2: 表格内容抽取
**作为** 用户  
**我想要** 系统能够识别并保留表格结构  
**以便** 问答时能正确回答表格中的数据问题

**验收标准：**
- [ ] 识别 PDF 中的表格（边框表格 + 无线条表格）
- [ ] 表格转为 Markdown 格式存储
- [ ] 保留表格表头和行列结构
- [ ] 问答时能正确回答"第N行第M列是什么"

**技术方案：**
- pdfplumber 或 tabula-java 表格抽取
- 输出 Markdown 表格格式

---

### US-2.3: 文档层级结构识别
**作为** 用户  
**我想要** 系统能识别文档的章节层级  
**以便** 检索时保留上下文关系

**验收标准：**
- [ ] 识别文档标题层级（H1-H6）
- [ ] 识别段落、列表、引用块
- [ ] 分块时保留同章节内容在一起
- [ ] 检索结果附带来源章节信息

---

## 2. 分块策略优化

### US-2.4: 语义分块策略
**作为** 开发者  
**我想要** 实现基于语义的智能分块  
**以便** 每个块都是完整语义单元

**验收标准：**
- [ ] 基于句子/段落的自然断点分块
- [ ] 避免在句子中间切断
- [ ] 相似主题内容分在同一块
- [ ] 可配置块大小（min/max）
- [ ] 支持配置项切换：fixed / semantic

---

### US-2.5: 递归分块策略
**作为** 开发者  
**我想要** 实现递归字符分块  
**以便** 大文档能被安全处理

**验收标准：**
- [ ] 优先按段落分块
- [ ] 段落过大时递归按句子分
- [ ] 句子还过大时按字符数硬切
- [ ] 保留块之间的 overlap（重叠50-100字符）
- [ ] 边界保持语义完整性

---

### US-2.6: 按章节分块
**作为** 用户  
**我想要** 按文档原有章节结构分块  
**以便** 检索结果更易理解

**验收标准：**
- [ ] 自动识别文档章节标题
- [ ] 以章节为粒度分块
- [ ] 块元数据包含所属章节
- [ ] API 返回结果包含章节信息

---

## 3. 多 Embedding 模型支持

### US-2.7: Embedding 模型配置化
**作为** 开发者  
**我想要** Embedding 模型可配置切换  
**以便** 根据效果和成本选择模型

**验收标准：**
- [ ] 环境变量配置 `EMBEDDING_MODEL`
- [ ] 支持本地 Ollama 模型
- [ ] 支持云服务（OpenAI、SiliconFlow 等）
- [ ] 模型配置热切换
- [ ] Embedding 生成失败时的降级策略

**配置示例：**
```properties
# Ollama 本地
spring.ai.ollama.embedding.options.model=qwen3-embedding:4b

# SiliconFlow
spring.ai.openai.api-key=${SILICONFLOW_API_KEY}
spring.ai.openai.base-url=https://api.siliconflow.cn/v1
```

---

### US-2.8: Embedding 模型对比评测
**作为** 项目维护者  
**我想要** 知道不同 Embedding 模型的效果差异  
**以便** 选择最优模型

**验收标准：**
- [ ] 标准化评测数据集（100条问答对）
- [ ] 评测脚本对比多模型效果
- [ ] 输出召回率、M RR @K 指标
- [ ] 评测报告（Markdown 格式）
- [ ] 评测结果存档

---

## 4. 混合检索

### US-2.9: BM25 关键词检索
**作为** 用户  
**我想要** 系统结合向量和关键词检索  
**以便** 专有名词/术语查询更准确

**验收标准：**
- [ ] 实现 BM25 算法（或集成 Lucene）
- [ ] Chroma 向量检索 + BM25 融合
- [ ] 配置权重比例（默认 7:3）
- [ ] 可配置启用/禁用混合检索
- [ ] 检索耗时增加 < 50ms

---

### US-2.10: 检索结果融合
**作为** 开发者  
**我想要** 向量检索和关键词检索结果融合排序  
**以便** 综合两者优势

**验收标准：**
- [ ] RRF（Reciprocal Rank Fusion）融合算法
- [ ] 或支持配置：加权求和
- [ ] 融合后返回 Top-K 结果
- [ ] 日志记录各检索方式得分

---

## 5. 重排序（Rerank）

### US-2.11: Cross-Encoder 重排序
**作为** 用户  
**我想要** 检索结果经过重排序  
**以便** 最相关的答案排在最前面

**验收标准：**
- [ ] 集成 Cross-Encoder 模型（如：BAAI/bge-reranker）
- [ ] 向量检索初筛 Top-20
- [ ] Cross-Encoder 重排为 Top-5
- [ ] 重排耗时 < 200ms
- [ ] 配置项控制重排启用/禁用

---

### US-2.12: 重排模型配置化
**作为** 开发者  
**我想要** 重排模型可切换  
**以便** 根据场景选择模型

**验收标准：**
- [ ] 支持 Ollama 本地部署
- [ ] 支持 SiliconFlow 等云服务
- [ ] 环境变量配置 `RERANK_MODEL`
- [ ] 模型加载失败时的降级处理

## 6. RAG 效果评估

### US-2.13: RAG 评测指标计算
**作为** 项目维护者  
**我想要** 系统能自动评估 RAG 效果  
**以便** 量化优化效果

**验收标准：**
- [ ] Faithfulness（回答忠实度）
- [ ] Answer Relevancy（答案相关性）
- [ ] Context Precision（上下文精确度）
- [ ] Context Recall（上下文召回率）
- [ ] 每轮问答自动计算并记录

---

### US-2.14: 评测数据集管理
**作为** 开发者  
**我想要** 管理评测用的问答数据集  
**以便** 持续跟踪效果

**验收标准：**
- [ ] 评测数据集 JSON 格式存储
- [ ] 支持批量导入/导出
- [ ] 每条数据包含：question, ground_truth, context
- [ ] 评测结果持久化
- [ ] 评测历史对比图表

---

### US-2.15: 评测报告生成
**作为** 项目维护者  
**我想要** 自动生成评测报告  
**以便** 展示优化效果

**验收标准：**
- [ ] 自动计算平均指标
- [ ] 生成 Markdown 报告
- [ ] 包含改进对比（如果有历史数据）
- [ ] CI 中自动运行评测
- [ ] 报告推送到 GitHub Pages

## 7. 检索增强

### US-2.16: Query 改写
**作为** 用户  
**我想要** 系统能理解问题的真实意图  
**以便** 歧义问题也能正确回答

**验收标准：**
- [ ] 识别代词指代（如"它"指代上文内容）
- [ ] 同义词扩展（用户说"电脑"→"计算机"）
- [ ] 查询改写后记录到日志
- [ ] 配置项控制启用/禁用

---

### US-2.17: HyDE 查询增强
**作为** 用户  
**我想要** 系统能生成假设性答案来辅助检索  
**以便** 模糊问题也能找到相关内容

**验收标准：**
- [ ] LLM 先生成假设性答案
- [ ] 用假设答案去向量检索
- [ ] 原始问题和假设答案对比选择
- [ ] 关闭时不影响原有逻辑

---

### US-2.18: 多跳推理
**作为** 用户  
**我想要** 系统能进行多步推理  
**以便** 回答需要综合多处信息的问题

**验收标准：**
- [ ] 支持 2-3 跳推理
- [ ] 中间结果可视化
- [ ] 推理链可追溯
- [ ] 复杂度自动降级（简单问题不走推理）

---
